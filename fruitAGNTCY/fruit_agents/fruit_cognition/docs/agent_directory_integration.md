# Agent Directory Integration Guide

This guide demonstrates how to integrate **fruitCognition fruit_cognition** with **[agntcy dir](https://github.com/agntcy/dir)** for automated agent discovery and management.

Stack versions follow **[AGNTCY Directory v1.0.0](https://docs.agntcy.org/dir/getting-started/)** (`ghcr.io/agntcy/dir-apiserver:v1.0.0`, `ghcr.io/agntcy/dir-ctl:v1.0.0`, PostgreSQL for metadata, Zot `v2.1.15` for OCI storage).

## Overview

The integration enables automatic translation and publication of all fruit_cognition agent A2A cards to a local agntcy directory service.

## Quick Start

### 1. Start the Directory Service

From the **fruit_cognition** project directory, launch Postgres, the directory API server, MCP server (`dirctl`), and the OCI registry (all use the `recruiter` compose profile):

```bash
docker compose --profile recruiter up -d postgres zot dir-api-server dir-mcp-server
```

This starts:

- `postgres`: PostgreSQL backing store for the directory API server
- `dir-api-server`: Directory API (`ghcr.io/agntcy/dir-apiserver:v1.0.0`)
- `dir-mcp-server`: MCP bridge (`ghcr.io/agntcy/dir-ctl:v1.0.0`, `mcp serve` with open stdin)
- `zot`: OCI-compliant registry (`ghcr.io/project-zot/zot:v2.1.15`) for agent artifacts

To stop, use the same service list for `down`, e.g. `docker compose --profile recruiter down` or `docker compose --profile '*' down`.
A plain `docker compose down` can fail with the "network still in use" message if the started containers have profiles defined.

### 2. Install Development Dependencies

Install the required development dependencies for the integration scripts:

```bash
uv sync --extra dev
```

### 3. Publish Agent Records

Run the automated agent record publication script:

```bash
./scripts/publish_agent_records.sh
```

This script scans the `agents/` directory and publishes agent records. The corresponding JSON files for each agent will be stored in the appropriate `supervisors` subfolder (e.g., `agents/supervisors/auction/oasf/agents/`). **Do not modify these JSON files directly**—they are served as static files via APIs, and are intentionally kept static because publishing and processing records can be time-consuming at startup or runtime. Our goal is to showcase the record publishing process; if you want to experiment, feel free to manually run the instructions above.

> **Note:** MCP server JSONs are not generated by this script—they are manually added by us. Automated support for MCP JSON generation is not available yet.

This script will:

- Scan the `agents/` directory for A2A card definitions
- Convert agent metadata to OASF format
- Upload records to the running directory service
- Generate content identifiers (CIDs) for published records

### 4. Verify and Interact with Records

After publishing, you can:

1. **View published records**: Check the generated `published_cids.json` file for CIDs of your published agents
2. **Interact with directory**: Use the `dirctl` CLI tool (v1.0.0) to query and manage directory records
